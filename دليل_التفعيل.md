# دليل تفعيل نظام الدخول (أدمن / موظفين) — خطوة بخطوة

النظام دلوقتي بقى فيه 3 صفحات رئيسية:
- **login.html** — صفحة تسجيل الدخول (نقطة البداية للجميع)
- **index.html** — لوحة المدير/الأدمن (كل النماذج + صفحة "متابعة الموظفين" الجديدة)
- **employee.html** — بوابة الموظف (مشاريعه، إنجازاته، طلباته)

كل البيانات (الحسابات، المشاريع، الإنجازات، الطلبات) هتتخزن على **Firebase** (خدمة مجانية من Google) عشان تكون متاحة لأي حد يدخل من أي جهاز.

---

## 1) إنشاء مشروع Firebase (5 دقائق)

1. روح على https://console.firebase.google.com وسجّل دخول بحساب Google.
2. اضغط **"إضافة مشروع / Add project"** واختار اسم زي `tekgo-hr`.
3. تقدر توقف Google Analytics (مش لازمة).

## 2) تفعيل تسجيل الدخول بالبريد وكلمة المرور

1. من القائمة الجانبية: **Build → Authentication → Get started**.
2. من تبويب **Sign-in method**، فعّل **Email/Password**.

## 3) إنشاء قاعدة بيانات Firestore

1. من القائمة الجانبية: **Build → Firestore Database → Create database**.
2. اختار **Start in production mode** ثم اختار أقرب منطقة لك (مثلاً `eur3` أو `europe-west`).
3. بعد الإنشاء، افتح تبويب **Rules**، وامسح المحتوى الموجود، والصق بدلاً منه **كل محتوى ملف `firestore.rules.txt`** المرفق، ثم اضغط **Publish**.

## 4) ربط الموقع بمشروعك (خطوة مهمة جداً)

1. من إعدادات المشروع ⚙️ (بجانب "Project Overview") → **Project settings**.
2. تحت "Your apps"، اضغط أيقونة الويب `</>` وسمّي التطبيق أي اسم، ثم **Register app**.
3. هيظهرلك كود فيه `firebaseConfig = {...}` — انسخ القيم دي (apiKey, authDomain, projectId... إلخ).
4. افتح ملف **firebase-config.js** من ملفات النظام، واستبدل القيم الوهمية بالقيم الحقيقية اللي نسختها.

## 5) إنشاء أول حساب أدمن (المدير)

**الطريقة السهلة (موصى بها):**

1. بعد ما تخلّص خطوة (4) وتربط الموقع بمشروعك، افتح **setup.html** (أو افتح `login.html` وهتلاقي رابط "أول مرة تستخدم النظام؟ إعداد حساب المدير الأول").
2. املأ الاسم، البريد الإلكتروني، وكلمة المرور، واضغط **"إنشاء حساب المدير والدخول"**.
3. هيتم إنشاء حساب الدخول وملف المستخدم بدور `admin` تلقائياً، وتتحول مباشرة على لوحة الأدمن.

> هذه الصفحة شغّالة **مرة واحدة بس**: أول ما يتعمل أول حساب أدمن، أي محاولة تانية لفتح `setup.html` هتظهرلك رسالة "النظام مُعَد بالفعل" وهتوجّهك لصفحة الدخول. القواعد الأمنية (Rules) بتمنع إنشاء أدمن تاني من غير ما تكون مسجّل دخول كأدمن بالفعل.

**الطريقة اليدوية (بديل، لو حبيت):**

1. **Authentication → Users → Add user** → اكتب بريدك الإلكتروني وكلمة مرور واضغط Add user.
2. هيظهرلك المستخدم في القائمة ومعاه **User UID** طويل — انسخه.
3. روح **Firestore Database → Start collection** (لو أول مرة) أو **Add document** جوه collection موجود:
   - اسم الـ Collection: `users`
   - Document ID: الصق الـ **UID** اللي نسخته
   - أضف الحقول دي بالظبط:
     | الحقل | النوع | القيمة |
     |---|---|---|
     | name | string | اسمك |
     | email | string | نفس البريد اللي سجّلته |
     | role | string | `admin` |
4. لازم كمان تعمل document في collection اسمها `system` بـ Document ID: `meta` وفيه حقل `setupDone` (boolean) = `true` — وإلا صفحة `setup.html` هتفضل شغّالة وممكن حد يعمل بيها حساب أدمن تاني.
5. احفظ. دلوقتي افتح **login.html** وسجّل دخول بنفس البريد وكلمة المرور — هتدخل مباشرة على لوحة الأدمن.

## 6) إضافة الموظفين

من داخل لوحة الأدمن → **متابعة الموظفين** → اكتب اسم الموظف، بريده الإلكتروني، وكلمة مرور مبدئية، واضغط **إنشاء الحساب**. الموظف بعد كده يدخل من **login.html** بنفس البيانات، ويقدر يغيّر كلمة المرور بنفسه لاحقاً من رابط "نسيت كلمة المرور".

## 7) إضافة مشاريع للموظفين

من داخل لوحة الأدمن → **إدارة المشاريع** (في قسم "متابعة الأداء" بالقائمة الجانبية):
1. اكتب عنوان المشروع ووصفاً مختصراً له.
2. حدد من قائمة الموظفين مين المسؤول عن المشروع (تقدر تختار أكتر من موظف).
3. اضغط **"➕ إنشاء المشروع"** — كده المشروع هيظهر فوراً في قائمة "المشاريع الحالية"، وهيظهر تلقائياً لكل موظف مُعيَّن عليه في بوابته الخاصة (employee.html) عشان يحدّث نسبة تقدّمه فيه.

تقدر كمان من نفس الصفحة:
- **تعديل مشروع** (العنوان، الوصف، أو الموظفين المُعيَّنين) بالضغط على "✏️ تعديل المشروع".
- **حذف مشروع** نهائياً بالضغط على "🗑 حذف المشروع".
- متابعة نسبة تقدّم كل موظف في كل مشروع (شريط تقدّم + الحالة + أي ملاحظة كتبها الموظف)، وده نفس اللي بيظهر في صفحة "متابعة الموظفين" لكن مُجمَّع حسب المشروع بدل الموظف.

> ملحوظة: لو كنت أضفت مشاريع يدوياً من قبل عن طريق Firestore Console، هتظهر تلقائياً في نفس الصفحة زي أي مشروع تاني، وتقدر تعدّل عليها أو تحذفها بنفس الطريقة.

## 8) التقارير الأسبوعية للموظفين

كل موظف من بوابته (`employee.html`) عنده تبويب **"📆 تقاريري الأسبوعية"** يقدر منه يرسل تقريراً أسبوعياً (بداية الأسبوع + ملخص المهام والتقدم والمعوقات). التقارير دي تظهر تلقائياً للأدمن من لوحة **"متابعة الموظفين"** — كل موظف هتلاقي تحته قسم "📆 التقارير الأسبوعية" بكل التقارير اللي أرسلها، بالإضافة لعدّاد سريع في أعلى كارت الموظف. مفيش أي إعداد إضافي مطلوب — الميزة دي شغّالة تلقائياً بمجرد تحديث ملفات `firestore.rules.txt` (لو كان النظام شغّال عندك من قبل، راجع الملحوظة في نهاية الدليل).

---

## ملحوظة لو النظام كان شغّال عندك من قبل (تحديث)

لو أنت أصلاً عندك أدمن شغّال وبتحدّث ملفات النظام فقط (مش أول مرة)، لازم:
1. تلصق محتوى `firestore.rules.txt` الجديد في **Rules** وتضغط **Publish** تاني (القواعد بتتحدّث مع كل ميزة جديدة — آخر تحديث أضاف صلاحيات collection اسمها `weeklyReports`).
2. تعمل document واحد يدوي بس (مش أكتر): Collection `system` → Document ID `meta` → حقل `setupDone` (boolean) = `true`. ده عشان تقفل صفحة `setup.html` وتمنع أي حد من عمل أدمن تاني بيها.

## أهم نقطة تتأكد منها لو حصلت مشكلة

- لو الصفحة فضلت "بيضا" أو فيها لودينج مايخلصش: يبقى غالباً `firebase-config.js` لسه فيه بيانات وهمية، أو الـ Authentication/Firestore لسه مش متفعّلين.
- لو ظهر خطأ "Missing or insufficient permissions": يبقى قواعد الأمان (Rules) لسه مانُشرتش، ارجع لخطوة (3).
- كل حاجة شغالة على الباقة المجانية (Spark) من Firebase وكافية جداً لعدد موظفين متوسط.
